# 선언형 모드 애드온

*English: [mode-addons.md](mode-addons.md) · 한국어: 이 문서*

Remote AI Coder는 다섯 가지 **빌트인 모드**(`agent`, `plan`, `ask`, `research`, fix 흐름)를 기본 제공합니다. 코드를 전혀 바꾸지 않고도 사용자가 직접 **읽기 전용 프롬프트 프리셋 모드**를 추가할 수 있습니다. 애드온 디렉터리에 YAML 파일을 넣고 서버를 재시작하면 됩니다. 각 애드온은 새 트리거(슬래시 명령과 선택적 자연어 접두)를 등록하고, 선택된 AI CLI를 고정된 프롬프트 프리픽스와 함께 detached worktree에서 실행합니다.

애드온은 **반드시 읽기 전용**입니다. 쓰기·커밋·push 권한은 빌트인 전용으로 남아 있어, 드롭한 YAML이 쓰기 권한을 부여할 수 없습니다.

## YAML 파일을 두는 위치

파일 1개가 모드 1개입니다. 아래 경로에 파일을 둡니다.

```text
~/.remote-coder/addons/*.yaml
```

이 디렉터리는 `REMOTE_CODER_HOME`(기본값 `~/.remote-coder`) 하위에 있습니다. 디렉터리가 없으면 애드온이 로드되지 않으며, 빌트인 모드에는 영향이 없습니다.

애드온은 **서버 부팅 시 1회** 로드됩니다. 핫리로드는 없습니다 — 파일을 추가·수정·삭제한 뒤에는 서버를 재시작해야 변경이 반영됩니다.

## YAML 스키마

파일은 다음 필드를 가진 단일 최상위 매핑입니다.

| 필드 | 필수 | 타입 | 설명 |
|---|---|---|---|
| `name` | 예 | 문자열 | 모드 식별자. 정규식 `^[a-z][a-z0-9_]{1,30}$`를 만족해야 합니다(소문자로 시작, 소문자/숫자/밑줄, 2~31자). `/name` 슬래시 명령에 사용됩니다. |
| `read_only` | 예 | 불리언 | 반드시 `true`여야 합니다. 그 외 값(`false`, `1`, `"true"` 포함)이면 파일이 거부됩니다. |
| `prompt` | 예 | 문자열 | 사용자 지시문 앞에 붙는 프리픽스 텍스트. 줄바꿈을 포함할 수 있습니다. |
| `slash` | 아니오 | 불리언 | 기본값 `true`. `true`면 `/name` 슬래시 명령으로 트리거됩니다. |
| `aliases` | 아니오 | 문자열 목록 | 자연어 접두 별칭(예: 한글 단어). 각 별칭은 콜론 접두 `별칭: 지시문` 형태로만 트리거됩니다. |
| `help` | 아니오 | 맵(언어 → 문자열) | 도움말 텍스트. 주어지면 `en` 키가 필수입니다. 슬래시 자동완성 설명에 사용됩니다. |
| `label` | 아니오 | 맵(언어 → 문자열) | 표시 라벨. 생략하면 `name`을 사용합니다. |

참고:

- `prompt` 텍스트는 권한에 영향을 주지 않습니다. "edit files"나 "push to remote" 같은 문구가 있어도 권한은 오직 `read_only` 플래그가 결정하며, 애드온 모드는 항상 읽기 전용입니다.
- `name`은 빌트인 이름(`agent`, `plan`, `ask`, `research`, `agent_fix`)과 충돌하면 안 됩니다.
- 어떤 트리거도 기존 트리거와 충돌하면 안 됩니다. 빌트인 슬래시 이름과 별칭(`plan`, `ask`, `research`, `계획`, `질문`, `조사`, `fix`, `수정`)은 이미 사용 중이며, 애드온이 이를 가릴 수 없습니다.
- YAML은 `yes`/`on`을 불리언 `true`로 파싱하므로 `read_only`에 쓸 수 있지만, 명확하게 `true`를 권장합니다.

## 완전한 예제

`~/.remote-coder/addons/review.yaml`:

```yaml
name: review
read_only: true
prompt: |
  You are in REVIEW mode. Read the code and report issues:
  correctness, security, readability, and tests.
  Do not modify files.
slash: true
aliases:
  - 리뷰
help:
  en: "Review the code and report issues"
  ko: "코드를 검토하고 문제를 보고"
label:
  en: "Review"
  ko: "리뷰"
```

서버를 재시작하면 `review` 모드가 등록되고, Telegram 슬래시 자동완성 메뉴(`setMyCommands`)에 `/review`가 추가됩니다.

## 모드 트리거 사용법

위 `review` 예제를 기준으로, 빌트인 읽기 전용 모드와 동일하게 세 가지 형태가 동작합니다.

- **지시문이 포함된 슬래시 명령**: `/review 로그인 흐름의 문제를 점검해줘`
- **자연어 별칭 접두**: `리뷰: 로그인 흐름의 문제를 점검해줘`
- **본문 없는 슬래시 명령**: `/review`만 보내면, 다음에 보내는 메시지가 지시문이 됩니다.

슬래시 명령은 영문 `name`만 사용합니다. 별칭은 콜론 접두(`별칭: 지시문`)로만 동작하며 `/별칭` 슬래시로는 동작하지 않습니다.

## 보안 제약

- **읽기 전용 전용**: 애드온 모드는 항상 detached worktree에서 커밋·push 없이 실행됩니다. `read_only: true`가 필수이며, 쓰기 가능한 스펙은 거부됩니다. 쓰기·커밋·push 모드는 빌트인 전용으로 남습니다.
- **프롬프트 텍스트는 권한이 아님**: `prompt`의 문구는 쓰기 권한을 부여하지 않습니다. 권한은 오직 읽기 전용 불변식으로 결정됩니다.

## 잘못된 파일은 건너뜀

검증에 실패한 파일 — 잘못된 `name`이나 스키마, `read_only`가 `true`가 아님, 빌트인/트리거 충돌, 필수 필드 누락, YAML 파싱 에러, 읽을 수 없는 파일 — 은 **경고 로그를 남기고 건너뜁니다.** 다른 유효한 애드온이나 서버 부팅에는 영향을 주지 않습니다(파일 단위 graceful skip). 경고 로그에는 짧은 사유 토큰만 남으며, 파일 경로와 내용은 기록되지 않습니다.
