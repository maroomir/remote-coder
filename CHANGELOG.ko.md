# 변경 이력

*English: [CHANGELOG.md](CHANGELOG.md) · 한국어: 이 문서*

이 파일은 [Keep a Changelog](https://keepachangelog.com/ko/1.1.0/) 형식을 따르며, 버전 번호는 [Semantic Versioning](https://semver.org/lang/ko/)을 따릅니다.

히스토리를 한 번에 정리할 때 Git 로그와 문서를 함께 보고, 기능 단위로 묶는 편이 좋습니다. 단계적으로 주제를 나누고 중복을 제거하려면 Cursor의 **Sequential Thinking** MCP(`sequentialthinking`)처럼 생각을 번호 매겨 기록하는 도구도 도움이 됩니다. 다만 그 도구는 저장소와 연동되어 자동 취합해 주지 않으므로, **원본은 항상 `git log`·릴리즈 태그·이 파일의 이전 버전**을 기준으로 삼는 것이 안전합니다.

## [미배포]

### 추가됨

- **봇 토큰 OS 키링 저장**: 봇 토큰과 webhook secret을 평문 `projects.json` 대신 OS 키링(macOS Keychain / Linux Secret Service / Windows Credential Locker)에 저장합니다. 키링이 있는 환경에서 처음 실행하면 기존 평문 시크릿을 키링으로 1회 이전하고(`projects.json.pre-keyring.bak`로 백업), 파일에서 제거합니다. 사용 가능한 키링 백엔드가 없으면 기존 평문 파일은 그대로 동작하며(또는 `REMOTE_CODER_ALLOW_PLAINTEXT_SECRETS=1` 옵트인), 그 외 신규 설치에서는 안내와 함께 부팅을 중단(fail-closed)합니다.
- **완료 알림의 diff/리스크 리뷰 카드**: 완료된 AGENT 또는 fix 작업 알림에 파일별 변경 요약(추가/삭제 줄 수)을 임팩트 순으로 표시하고, 고신뢰 리스크 항목(의존성 lockfile, DB 마이그레이션, 대량 삭제, 공유 유틸리티)을 강조해 모바일에서 변경 사항을 검토하기 쉽게 했습니다.

## [0.5.4] — 2026-06-22

### 추가됨

- **Ollama 로컬 모델 러너**: 기존 클라우드/CLI 러너와 함께 Ollama 기반 로컬 모델을 사용할 수 있으며, 모델 선택·진단·모니터링 출력·관리 UI 설정이 Telegram과 대시보드 흐름에 통합되었습니다.
- **선언형 모드 애드온**: `~/.remote-coder/addons/`에 YAML 파일을 넣으면 코드 변경 없이 새 읽기 전용 프롬프트 프리셋 모드(슬래시 명령과 자연어 접두)를 추가할 수 있습니다. 애드온은 읽기 전용 전용이며 부팅 시 1회 로드됩니다. [docs/mode-addons.ko.md](docs/mode-addons.ko.md)를 참고하세요.

### 변경됨

- **모드 처리를 레지스트리 중심으로 통합**: 기본 AGENT·ASK·PLAN·RESEARCH 동작이 공통 모드 레지스트리를 거치도록 바뀌어, Telegram 파싱·AI 프롬프트·읽기 전용 판정·애드온 검증이 하나의 원본을 사용합니다.
- **Ollama와 애드온 문서 보강**: README와 러너 문서에 Ollama 설정, 로컬 모델 동작, 선언형 모드 애드온 설계를 반영했습니다.

### 수정됨

- **민감한 Telegram 데이터 로그 노출 방지**: Webhook 등록 오류에 봇 토큰이 남지 않게 하고, webhook secret 비교를 constant-time으로 수행하며, `projects.json` 권한을 소유자 전용으로 제한했습니다.
- **Ollama 패치 처리 안전성 강화**: Ollama가 생성한 패치에서 안전하지 않은 경로를 적용 전에 거부합니다.
- **Job 메타데이터 정확도 개선**: 러너 토큰 사용량을 중복 집계하지 않고, Git 변경 수집에서 rename의 삭제 원본 대신 최종 대상 경로를 보고합니다.

## [0.5.3] — 2026-06-20

### 추가됨

- **Job 상태에서 전체 AI 출력 보기**: `/status <job_id>` 상세 패널에 **전체 로그 보기** 버튼을 추가해, 꼬리 일부가 아닌 AI stdout 전체를 페이지로 나눈 Telegram 메시지로 전달합니다.
- **로그 명령 선택 UI**: 인자 없이 `/log`를 호출하면 최근 Job을 인라인 버튼으로 표시하고, `/log`를 Telegram 슬래시 자동완성과 명령어 메뉴에 등록합니다.
- **완료 Job 리베이스 바로가기**: 성공한 Job 메시지에서 완료된 작업에 대한 rebase 액션을 표시할 수 있어, 다음 유지보수 작업을 Telegram에서 바로 시작할 수 있습니다.

### 수정됨

- **재시작 시 명령어 메뉴 동기화 유지**: 서버가 시작될 때마다 Telegram 명령어 메뉴(햄버거 버튼)와 슬래시 자동완성을 다시 등록하여, 나중에 추가된 명령어·모드(예: RESEARCH)가 `remote-coder up`을 다시 실행하지 않아도 노출됩니다.
- **Job 복구 경로**: 서버 시작 시 끝나지 않은 Job을 복구하고, fix Job 복구 라우팅을 되살려 중단된 작업을 더 안정적으로 이어갈 수 있습니다.

## [0.5.2] — 2026-06-15

### 추가됨

- **읽기 전용 RESEARCH 모드**: `/research`, `research:`, `조사:` 요청이 저장소 맥락과 인터넷 검색 안내를 선택한 AI CLI에 제공하고 detached worktree에서 실행됩니다. 커밋이나 브랜치 push는 생성하지 않습니다.

### 변경됨

- **명확한 모드 안내**: `/start`가 AGENT·PLAN·ASK·RESEARCH 액션을 간결한 레이아웃으로 표시하고, 모드 도움말에서 제거된 텍스트 확인 흐름 안내를 없앴습니다.
- **실행 가능한 GitHub CLI 설정 안내**: Telegram PR 흐름, CLI 진단, 관리 대시보드에서 `gh`가 없을 때 설치와 인증 방법을 안내합니다.
- **반복 가능한 릴리스 배포**: 릴리스 자동화가 기존 GitHub Release를 안전하게 재생성하고, 같은 버전이 이미 PyPI에 배포됐으면 업로드를 건너뛰어 workflow 재실행을 안정적으로 처리합니다.

## [0.5.1] — 2026-06-14

### 추가됨

- **Telegram 작업 상태 반응**: 작업을 접수하면 원본 요청 메시지에 반응을 남기고, 성공·실패·취소 결과에 따라 반응을 갱신합니다. 반응 전송 실패는 작업 처리에 영향을 주지 않습니다.
- **고정 명령 메뉴와 액션 색상**: 프로젝트 봇이 메시지 입력창 옆에 Telegram 슬래시 명령 메뉴를 고정하고, 주요·성공·위험 액션에 Bot API 버튼 스타일을 적용합니다.

### 변경됨

- **작업 결과 메시지 제자리 갱신**: 한 Telegram 메시지에 들어가는 결과는 작업 접수·하트비트 메시지를 제자리에서 교체합니다. 긴 결과나 수정 실패 시에는 오래된 중단 버튼을 제거한 뒤 별도 메시지로 폴백합니다.
- **간결한 채팅 안내**: `/start`, `/help`, 모니터링 출력을 더 짧고 읽기 쉬운 섹션과 목록으로 정리했습니다. 도움말 버튼은 설명 전용이며 명령을 실행하지 않습니다.

### 수정됨

- **논블로킹 시작 작업**: 저장소 pull과 Telegram 시작 알림을 HTTP 준비 완료 후 실행하고, 백그라운드 실패는 애플리케이션 종료를 깨뜨리지 않고 로그에 남깁니다.
- **Claude 답장 세션 폴백**: 새 worktree에서 실행되는 ASK·PLAN 및 기타 작업은 Claude CLI가 찾을 수 없는 cwd 종속 `--resume` 토큰을 더 이상 전달하지 않습니다. 논리적 Session ID와 주입된 답장 컨텍스트는 유지하며, 기존 linked worktree를 재사용할 때는 네이티브 세션 재개를 계속 사용합니다.

## [0.5.0] — 2026-06-13

### 추가

- **PLAN 모드 인라인 결정**: `/plan` 요청이 사용자만 정할 수 있는 선택지에 의존할 때, 모델이 이를 먼저 구조화된 블록으로 제시하고 봇이 각 결정을 Telegram 인라인 버튼으로(한 번에 하나씩) 물은 뒤 최종 계획을 만듭니다. 선택한 답은 2차 PLAN 실행에 주입되므로 Claude·Codex·Gemini에서 동일하게 동작합니다. 결정할 사항이 없으면 기존처럼 계획을 바로 전달합니다.
- **계획 실행 버튼**: PLAN 성공 결과에 **계획 실행** 인라인 버튼이 붙습니다. 한 번 누르면 승인된 계획을 구현하는 AGENT 작업이 제출되며(계획 본문과 원래 요청이 컨텍스트로 주입됨), 내용을 다시 입력하지 않고 계획에서 구현으로 넘어갈 수 있습니다.
- **작업 하트비트**: 작업 실행 중 "작업 접수" 메시지가 약 1분마다 경과 시간 줄(`⏳ 실행 중 (N분 경과)`)로 갱신되어, 긴 작업이 멈춘 것처럼 보이지 않습니다. 작업이 끝나면 메시지가 복원됩니다.

### 수정

- **Codex 커밋 메시지 생성**: 임시 디렉터리에서 Git 저장소 검사를 건너뛰어 Codex가 AI 커밋 메시지를 생성할 수 있게 했고, 실패 시 운영자 로그에 간결한 stderr 미리보기를 남깁니다.
- **Codex 세션 재개 옵션**: Codex 재개 작업에서 `--model`과 `--sandbox`를 `resume` 하위 명령 앞에 전달하도록 바꿔, 현재 CLI 파서에서 `unexpected argument '--sandbox'`로 실패하지 않게 했습니다.
- **범위가 명확하고 안정적인 `/pr` 흐름**: PR 후보를 현재 프로젝트·Telegram 채팅에서 성공한 Job의 원격 잔존 브랜치로 제한하고, 로컬에서 사라진 정상 브랜치도 표시합니다. 직접 입력에도 같은 소유권·원격 존재 검증을 적용하며, 커밋이 있는 성공 Job 알림에는 PR 버튼을 추가했습니다. 기존 PR URL을 재사용하고 `gh` 미설치·시간 초과·인증 실패를 webhook 예외 대신 실행 가능한 Telegram 오류로 반환합니다.
- **타임아웃·취소 시 부분 출력 보존**: 러너가 타임아웃이나 취소로 종료될 때, 그때까지 생성된 부분 stdout/stderr를 작업 로그에 저장하고 실패 알림에 요약으로 표시합니다. 기존에는 한 줄 오류만 남기고 전부 버려졌습니다.
- **Reply 연결 AI 세션**: Telegram 답장(reply)으로 연결된 작업들이 매 메시지마다 새 세션을 시작하지 않고 *동일한* AI CLI 세션을 이어갑니다. 각 reply 체인은 루트 메시지를 기준으로 묶이며 세션 id가 부여되고(SQLite에 저장, 잡 id와 연결), 러너가 네이티브로 세션을 재개합니다 — Claude는 `--session-id`/`--resume`, Codex는 `codex exec resume <id>`(rollout 파일에서 id 캡처), Gemini는 지원 시 `--resume <id>`, 미지원 시 기존 reply 컨텍스트 주입으로 폴백합니다. Claude 네이티브 재개는 체인이 동일 worktree를 재사용할 때(일반적인 브랜치 바인딩 reply 흐름) 동작합니다. `/clear memory`는 저장된 세션도 함께 삭제합니다.
- **Session ID 노출**: 사용자가 reply 체인이 같은 세션을 유지하는지 직접 확인할 수 있도록 세션 id를 표시합니다 — 작업 접수·결과 메시지(Job ID 바로 아래, 모노스페이스), `/status <job_id>` 상세, `/reports` 최근 Job 결과, 그리고 `/monitor memory`의 `Sessions:` 개수에 나타납니다.
- **간편 설정 위저드**: 관리 대시보드가 최초 사용자를 4단계 흐름(봇 토큰 확인 → Telegram 채팅 자동 감지 → 프로젝트 생성)으로 안내하여, 새 프로젝트를 약 1분 만에 연결할 수 있습니다. 로컬 전용 `POST /api/setup/validate-token`·`POST /api/setup/detect-chat` 엔드포인트가 이를 지원합니다.

### 변경

- **관리 UI 개편**: 모든 관리 페이지가 반응형 사이드바 대시보드 레이아웃과 단일 스타일시트(`admin.css`)의 현대적인 테마를 공유합니다. 홈은 대시보드로 바뀌었고, 프로젝트 등록·고급 설정·로그·데이터 조회는 새 셸 안에서 기존 동작을 유지합니다.
- **Telegram 메시지 스타일**: 봇 발신 메시지를 BotFather 스타일의 Telegram message entities로 렌더링합니다 — 제목·섹션 헤딩은 굵게, Job ID·브랜치·커밋 값은 모노스페이스로 표시되며 메시지 본문 텍스트는 그대로 유지됩니다.
- **채팅 패널 제자리 편집**: 메뉴·서브메뉴 버튼을 누르면 새 메시지 대신 기존 메시지를 수정하며, 액션에는 가벼운 토스트와 일관된 ‹ 뒤로 / ✖ 닫기 내비게이션이 붙습니다. `/status <job_id>` 상세에는 상태별 액션 버튼(중단, 또는 PR 올리기·리베이스)이 추가됩니다.
- **버튼 전용 확인**: 작업·수정·정리 확인은 항상 인라인 Yes/No 버튼을 사용합니다. `y`/`Y` 텍스트 경로와 관련 고급 설정은 제거되었습니다.
- **간소화된 고급 설정**: 거의 쓰이지 않던 5개 옵션(확인 버튼 토글, 서버 시작·중지 알림 토글, `/status` 최근 작업 수, 모호한 후속 요청 대화 수, reply 스니펫 최대 길이)을 제거하고 기존 기본값으로 고정했습니다. 기존 설정 파일은 로드 시 옛 키를 자동으로 제거합니다.

### 보안

- **무작위 webhook secret**: 관리 UI에서 secret을 비워 새 프로젝트를 만들면 공통 고정값 대신 프로젝트마다 고유한 256비트 URL-safe Telegram webhook secret을 자동 생성합니다.
- **프로젝트별 Job 직렬화**: 같은 프로젝트의 AGENT·PLAN·ASK·소스 수정 Job은 하나의 인메모리 프로젝트 잠금 아래 순서대로 실행하여 linked worktree의 동시 재사용·수정을 방지합니다. 서로 다른 프로젝트는 독립적으로 실행됩니다.

## [0.4.4] — 2026-06-10

## [0.1.0] — 2026-05-09

첫 PyPI·Git 릴리즈에 해당하는 기능 묶음입니다. 저장소 커밋 히스토리상의 초기 부팅부터 멀티봇·패키징까지의 누적 변경을 사용자 관점으로 요약합니다.

### 추가됨

- **FastAPI 서버·웹훅**: 텔레그램 Bot Webhook 수신, 빠른 응답 후 백그라운드 작업 흐름.
- **작업(Job) 관리**: 큐·상태(`queued`, `running`, `succeeded`, `failed`, `cancelled` 등), 타임아웃, 단계별 실패 기록, Job ID 고유성.
- **Git 워크트리 격리**: 메인 작업 디렉터리 대신 요청별 워크트리에서 AI 작업, 브랜치명 지정 또는 안전한 자동 생성, 변경 없으면 브랜치·커밋·푸시 생략.
- **AI 러너**: Claude Code, Codex CLI, Gemini CLI 공통 인터페이스·팩토리, 표준 출력·종료 코드·실행 시간 수집.
- **자동 커밋**: `type: title` 형식, 본문 불릿, `committed by remote-coder: <job-id>` 풋터; 필요 시 AI 기반 커밋 메시지 보강.
- **텔레그램 명령·파서**: 슬래시 명령, 자연어 요청 파싱, 자연어는 요약 확인 후 `y`/`Y`로만 Job 생성.
- **SQLite 대화·메모리**: 작업 결과 요약 저장; 답장(reply) 체인으로 이전 Job 맥락 연결.
- **`/init`**: 채팅의 모델 오버라이드·대기 중 확인 상태 초기화(SQLite·Git은 변경하지 않음).
- **인라인 키보드**: `/model`, `/status`, `/branch`, `/rebase`, `/stop` 등 인자 없이 호출 시 버튼 UI; 콜백은 기존 슬래시 명령 경로로 라우팅.
- **Git 보조 명령**: 브랜치 조회·전환, 리베이스·충돌 완화, 워크트리 정리(`clear`), 원격 브랜치 동기화(`/pull`), GitHub PR 생성(`/pr`), 모니터링용 브랜치·저장소 요약(`/monitor` 등).
- **관리자 UI**: 프로젝트·고급 설정 페이지, 인메모리 로그 버퍼 및 로그 뷰, DB 브라우징, 요약·통계·모델 사용량 표시.
- **모니터링·이벤트 로그**: 구조화 로깅, 텔레그램·작업·Git·러너 경계에서 `EventLogger` 사용; 메시지 본문에는 비밀·전체 경로 노출 금지 정책에 맞춘 미리보기만 기록.
- **토큰·모델 메타데이터**: Job 결과에 관측 모델명·토큰 사용량 저장; Codex/Claude/Gemini 사용량·쿼터 추적 보강.
- **설치 가능 패키지**: `pyproject.toml` 기반 빌드, 콘솔 진입점 `remote-coder`.
- **운영 문서**: 예시 환경 변수, 에이전트 규칙(`AGENTS.md`, `.clinerules/`, `.cursor/rules/`), 릴리즈 런북(`RELEASE.md`), Gemini 등 CLI 설정 가이드.
- **멀티봇·프로젝트 레지스트리**: 프로젝트별 봇 토큰·allowlist; 웹훅 경로는 봇 토큰 SHA-256 digest의 **앞 16자리 hex**로 라우팅(평문 토큰을 URL에 넣지 않음). `BotInstance` / `BotInstanceManager`로 인스턴스별 알림·인증·`CommandContext` 바인딩.
- **프로젝트별 상태**: Job·확인·모델 선호 등 채팅 상태를 봇에 묶인 프로젝트 기준으로 분리; 동일 사용자가 여러 봇과 대화해도 작업 목록이 섞이지 않도록 필터링.

### 변경됨

- **프로젝트 선택 UX**: 채팅에서 `/project`로 저장소를 바꾸는 흐름을 제거하고, **봇(웹훅) = 하나의 등록 프로젝트** 모델로 단순화. 프로젝트 목록·모니터링은 `/monitor` 등으로 정리.
- **`/start`·도움말**: 프로젝트 선택·버튼 배치 조정; `/help`는 텍스트 목록 위주로 단순화하는 등 UX 반복 개선.
- **커밋·알림 메시지**: 커밋 메시지 규칙·완료 알림 포맷 명확화; 모델 변경 확인 등 일부 메시지에서 불필요한 인라인 버튼 제거.
- **문서·규칙 동기화**: 보안(텔레그램·레지스트리 토큰 취급), Git/작업/러너 정책, 테스트·주석 최소화 정책 등을 `PLAN.md`와 규칙 파일에 반영.

### 수정됨

- 웹훅 등록 스크립트·콜백 처리·헬프·모델 명령 등 텔레그램 엣지 케이스 다수(예: 콜백 시 버튼 미표시, 알림 길이·전송 반복 문제).
- 리베이스·브랜치 클리어·Codex 샌드박스 등 Git·CLI 연동 관련 버그 수정.
- 읽기 전용 안내 문구, 변경 없는 작업 처리 등 사용자 오해를 줄이는 동작 수정.

### 보안·운영 참고

- Allowlist는 **등록 프로젝트(봇)별**. 레지스트리 파일의 토큰은 평문 저장 가능 설계이므로 파일 권한·유출 방지가 전제입니다.
- 사용자 메시지를 셸에 직접 넣지 않으며, 서브프로세스는 리스트 인자·명시적 `cwd`·타임아웃을 사용합니다.
